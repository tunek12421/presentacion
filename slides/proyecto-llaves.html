Proyecto 1

Sistema de Gestión de
Préstamo de Llaves

Sistema Web para Mejorar la Eficiencia en la Gestión del Préstamo de Llaves y Registro de Acceso Docente

Proyecto de Desarrollo de Software

El Problema

Proceso Manual
Registro en hojas de cálculo sin validación automática ni trazabilidad.
Error Humano
Precisión del 96-99% en entrada manual vs 99.9% en sistemas automatizados.
Sin Control de Acceso
Cualquier persona puede modificar los registros sin restricciones.
Sin Auditoría
Imposible determinar quién hizo cambios o cuándo ocurrieron.

La Solución

Sistema web con control de acceso por roles, validación automática de turnos y reconocimiento facial para el registro de préstamo de llaves.

Características

  • Registro automático de entrada/salida
  • Validación de turnos en tiempo real
  • Identificación por CI o reconocimiento facial
  • Historial completo y auditable

Tecnologías

  • Frontend: Angular
  • Backend: Go (Golang)
  • Base de Datos: PostgreSQL
  • Biometría: dlib (reconocimiento facial)

Roles del Sistema

Administrador
Gestión completa de usuarios, docentes, turnos y llaves.
Jefe de Carrera
Gestión de docentes y consulta de registros con exportación.
Bibliotecario / Becario
Registro de entrada/salida de llaves e identificación de docentes.
Docente
Solicita y devuelve llaves. Se identifica por CI o rostro.

Flujo: Registro de Entrada

Pasos

  1. Docente llega a biblioteca
  2. Se identifica (CI o rostro)
  3. Sistema valida identidad
  4. Sistema verifica turno asignado
  5. Bibliotecario busca llave
  6. Sistema verifica disponibilidad
  7. Se registra entrada
  8. Docente recibe llave física

Validaciones Automáticas

  • Docente registrado y activo
  • Turno asignado para el día
  • Hora dentro del rango del turno
  • Llave existe y está disponible
  • Llave no está en mantenimiento
Reconocimiento Facial

Opción alternativa al CI físico. Captura en tiempo real con webcam.

Funcionalidades Destacadas

Reconocimiento Facial

Identificación automática de docentes mediante webcam. Alternativa al CI físico para agilizar el proceso de registro.

Validación de Turnos

Verifica automáticamente si el docente tiene turno asignado en el horario y día actual antes de entregar la llave.

Control de Llaves

Estado en tiempo real de cada llave: disponible, en uso, en mantenimiento o perdida. Trazabilidad completa.

Auditoría Completa

Registro de todas las operaciones con fecha, hora y usuario responsable. Exportación de reportes en CSV.
