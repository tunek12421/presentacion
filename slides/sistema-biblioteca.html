Antecedentes

En la administración universitaria moderna, la automatización es clave para la eficiencia institucional. Tradicionalmente, la gestión de inventarios y préstamos en bibliotecas universitarias se realizaba mediante hojas de cálculo y registros manuales, lo que generaba silos de información, falta de trazabilidad y riesgos de pérdida o manipulación de datos. Según Oracle (2023), los métodos manuales carecen de integridad y seguridad, siendo inadecuados para activos institucionales.

Actualmente, la gestión de inventario y préstamos en la biblioteca de la UPDS depende de procesos manuales y herramientas ofimáticas básicas, sin validación automática ni control de roles.

Planteamiento del Problema

1. Ineficacia Manual
El registro manual consume tiempo y aumenta el riesgo de pérdida de información (Parchment, 2024).
2. Error Humano
La entrada manual tiene menor precisión frente a sistemas automatizados (DocuClipper, 2025).
3. Sin Control RBAC
No existe control de acceso basado en roles (IBM).
4. Sin Auditoría
Falta de registros de auditoría para la trazabilidad (NIST, 1997).

Situación Problemática

Problema Central

Ineficiencia en la gestión de inventario y préstamos de biblioteca.

Causas

  • Uso de herramientas no especializadas
  • Entrada manual sin validación
  • Ausencia de control de acceso por roles
  • Falta de trazabilidad

Efectos

  • Inconsistencia de datos
  • Saturación operativa
  • Vulnerabilidad de la información
  • Incertidumbre en auditoría

Pregunta de Investigación

¿De qué manera un sistema web mejora la eficiencia en la gestión de inventario y préstamos de biblioteca en la UPDS durante la gestión 2026?

Objetivo General

Desarrollar un sistema web para mejorar la eficiencia en la gestión de inventario y préstamos de biblioteca en la UPDS.

Objetivos Específicos

  1. Diagnosticar el proceso actual y sus deficiencias.
  2. Diseñar la arquitectura de software y base de datos relacional.
  3. Implementar módulos de gestión de usuarios, inventario y préstamos.
  4. Validar la integridad de los datos y la seguridad del sistema.
